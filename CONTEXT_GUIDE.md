# Guia de Persistência de Contexto do Projeto

Este documento orienta como manter o contexto do projeto entre sessões e garantir rastreabilidade das tarefas e modificações.

## Recomendações

- Utilize arquivos como PROJECT_STRUCTURE.md, GIT_USAGE.md e CHANGELOG.md para registrar estrutura, comandos e histórico.
- Salve anotações importantes sobre decisões técnicas e problemas encontrados em arquivos markdown.
- Use extensões do VSCode como "Project Manager" ou "Workspace Storage" para salvar o estado do workspace, abas abertas e histórico de navegação.
- Considere integrar ferramentas de automação (ex: Roo, scripts personalizados) para atualizar automaticamente o contexto e histórico após cada tarefa.
- Mantenha o README.md atualizado com instruções de uso e links para os arquivos de contexto.

## Recuperação de Contexto

- Ao iniciar uma nova sessão, consulte os arquivos de contexto para relembrar o estado do projeto.
- Compartilhe esses arquivos com colaboradores para garantir alinhamento e continuidade.
